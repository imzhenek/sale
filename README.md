# NOIRFRAME — Telegram Mini App модельного агентства

Каталог моделей, заявки на съёмку и админка живут внутри Telegram: без сайта,
без паролей и кодов — личность клиента и админа подтверждает сам Telegram
(подпись `initData`, HMAC-SHA256 по алгоритму Telegram).

## Как это устроено

- **Клиент** открывает бота → жмёт кнопку каталога (или значок меню рядом с полем ввода) →
  видит модели, заходит в карточку, отправляет заявку. Заявка тут же летит
  сообщением в вашу Telegram-группу с кнопками «Подтвердить/Отклонить».
- **Клиент** может посмотреть статус своих заявок в разделе «Мои заявки» —
  это просто фильтр по его Telegram ID, никакого логина не нужно.
- **Админ** (ты) пишет боту `/admin` → получает кнопку в админку. Доступ
  проверяется по списку `ADMIN_TELEGRAM_IDS` — открыть админку сможет
  только человек с этим Telegram ID, даже если ссылка утечёт.
- При нажатии «Подтвердить/Отклонить» (хоть в группе, хоть в админке) —
  клиенту прилетает личное сообщение от бота со статусом.

## Стек

Node.js + Express, better-sqlite3 (файловая БД, без внешнего сервера),
чистый JS без сборки на фронте (сами Mini App — обычные статичные HTML/JS,
которые Telegram открывает в WebView).

## Локальный запуск

```bash
npm install
cp .env.example .env   # заполнить TELEGRAM_BOT_TOKEN и остальное
npm start
```

Сервер поднимется на `http://localhost:3000`. Каталог (`GET /api/models`)
работает сразу, но заявки/админка требуют подписи Telegram — их из браузера
напрямую не подёргать, только через сам Telegram после деплоя.

## Деплой на Railway (как в твоих прошлых проектах)

1. Залить репозиторий на GitHub — **файлы должны быть в корне репозитория**,
   не во вложенной папке (та же грабля, что была с сайтом тату-салона).
2. В Railway: New Project → Deploy from GitHub repo.
3. В Variables добавить те же переменные, что в `.env.example`:
   `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `PUBLIC_URL`, `ADMIN_TELEGRAM_IDS`.
   `PUBLIC_URL` — это домен, который Railway выдаст после первого деплоя
   (`https://....up.railway.app`), без слеша на конце.
4. Start command можно не менять — Railway подхватит `npm start` из `package.json`.

## Настройка бота

1. Создать бота через [@BotFather](https://t.me/BotFather) → получить `TELEGRAM_BOT_TOKEN`.
2. Добавить бота в рабочую группу агентства (куда будут падать заявки),
   написать любое сообщение и открыть в браузере:
   `https://api.telegram.org/bot<TOKEN>/getUpdates` — найти `"chat":{"id": ...}`,
   это и есть `TELEGRAM_CHAT_ID` (для групп — отрицательное число).
3. Узнать свой Telegram ID через [@userinfobot](https://t.me/userinfobot) →
   вписать в `ADMIN_TELEGRAM_IDS` (через запятую, если админов несколько).
4. После деплоя и заполнения `PUBLIC_URL` в переменных Railway выполнить один раз:
   ```bash
   npm run bot-setup
   ```
   Это выставит боту постоянную кнопку меню («Каталог»), команды `/start` и `/admin`,
   и вебхук на приём сообщений и колбэков от кнопок.
5. Готово — пиши `/start` своему боту.

Если потом сменится домен (например, привяжешь свой), достаточно обновить
`PUBLIC_URL` и снова запустить `npm run bot-setup`.

## Добавление моделей

Заходишь в бота → `/admin` → «Открыть админку» → «Модели» → «+ Добавить».
Фото загружаются прямо из телефона (доступ к галерее даёт сам Telegram WebView).

**Важный нюанс:** загруженные фото сейчас хранятся на диске Railway
(`public/uploads`), а он не постоянный — при каждом передеплое (например,
после `git push`) файлы удаляются. Для MVP и тестирования это ок, но перед
тем как реально работать с агентством, стоит подключить облачное хранилище
(Cloudinary / S3-совместимое) — если понадобится, помогу это добавить.

## Структура проекта

```
server.js              — точка входа, роуты
db.js                   — SQLite: модели и заявки
telegram.js             — отправка уведомлений в Bot API
bot-setup.js             — разовая настройка кнопки меню/команд/вебхука
utils/telegram-auth.js   — проверка подписи initData
middleware/telegram-auth.js — express-мидлвары auth/admin
routes/api-public.js     — каталог, заявки, «мои заявки»
routes/api-admin.js      — CRUD моделей, управление заявками
routes/telegram-webhook.js — команды бота, кнопки подтверждения
public/miniapp/          — клиентский Mini App (каталог, заявка, кабинет)
public/admin-app/         — админский Mini App
public/shared.css         — общие стили (тёмная streetwear-эстетика)
```
